THE TIMELINE QUERY (D-06 / Pitfall 3 — correlated subqueries, positional IN-list, native prepared).
Bind order is positional: my_reaction `?` FIRST (it appears first in the SELECT list), then the
authors IN-list `?`s, then LIMIT `?`:
```sql
SELECT p.id, p.author_id, p.content, p.image_url, p.repost_of, p.created_at,
       (SELECT COUNT(*) FROM reactions r WHERE r.post_id = p.id)                      AS reaction_count,
       (SELECT COUNT(*) FROM comments  c WHERE c.post_id = p.id)                      AS comment_count,
       (SELECT r2.type  FROM reactions r2 WHERE r2.post_id = p.id AND r2.user_id = ?) AS my_reaction
  FROM posts p
 WHERE p.author_id IN (<?,?,...>)
 ORDER BY p.created_at DESC, p.id DESC
 LIMIT ?
```
Coerce: reaction_count/comment_count → (int); repost_of → int|null; my_reaction stays string|null.
NEVER `posts LEFT JOIN reactions LEFT JOIN comments ... GROUP BY` — that multiplies counts (fan-trap).

ROUTE SET (mirror connection-service ordering: literal `/posts` collection BEFORE parameterized;
numeric `{id:[0-9]+}` constraints so `/posts/{id}/comments` vs `/comments/{id}` never collide):
```php
$app->get   ('/health',                      [HealthController::class, 'health']);          // keep
$app->get   ('/posts',                       [PostController::class, 'index']);             // ?authors=&viewer=&limit= OR ?ids=

$app->post  ('/posts/{id:[0-9]+}/repost',    [PostController::class, 'repost']);
$app->post  ('/posts/{id:[0-9]+}/reactions', [PostController::class, 'react']);             // upsert {type}
$app->delete('/posts/{id:[0-9]+}/reactions', [PostController::class, 'unreact']);           // remove caller's
$app->get   ('/posts/{id:[0-9]+}/comments',  [CommentController::class, 'index']);
$app->post  ('/posts/{id:[0-9]+}/comments',  [CommentController::class, 'create']);

$app->delete('/comments/{id:[0-9]+}',        [CommentController::class, 'delete']);         // owner only
```
NOTE: register `/posts/{id}/comments`, `/posts/{id}/reactions`, `/posts/{id}/repost` (more specific)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gateway → feed-service (X-User-Id) | feed-service trusts X-User-Id set by gateway; no JWT, no host port (network-isolated) |
| request body → feed-service writes | author_id/user_id must come from X-User-Id header, NEVER the body (IDOR oracle) |
| viewer query param → my_reaction subquery | viewer scopes the my_reaction scalar subquery; viewer=0 → null (not another user's reaction) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Elevation/Tampering (IDOR) | PostController::delete, CommentController::delete | mitigate | Owner check `author_id == X-User-Id` → 403; caller from header only; uniform 404 when not found. |
| T-04-07 | Tampering | PostController::create/react/repost, CommentController::create | mitigate | author_id/user_id always from `getHeaderLine('X-User-Id')`, never the request body (verify grep). |
| T-04-08 | Tampering (SQLi, dynamic IN-list) | PostController::index timeline/batch | mitigate | Native prepared statements (EMULATE_PREPARES=false), positional placeholders for the IN-list, never string-interpolated ids. |
| T-04-09 | Integrity (count fan-trap) | PostController::index/show/find | mitigate | Correlated scalar subqueries per count; verify asserts NO `LEFT JOIN reactions/comments` (Pitfall 3). |
| T-04-10 | Tampering (double reaction) | reactions table + react() | mitigate | `UNIQUE(post_id,user_id)` + `ON DUPLICATE KEY UPDATE` upsert — at most one row per (post,user) (D-02). |
| T-04-11 | Integrity (orphan / missing-post) | react/comment/repost | mitigate | Existence invariant: react/comment/repost to a missing post → 404 (findPost/postExists null guard). |
| T-04-12 | Info disclosure (wrong viewer) | my_reaction subquery | mitigate | Subquery scoped `r2.user_id = :viewer`; viewer=0 returns NULL; type returned as string\|null, not bool (Pitfall 4). |
