THE COMPOSITION (04-RESEARCH Pattern 1, lines 144-223) — FeedController::feed:
  STEP 1 author universe: `$authorIds=[$me]`; try ConnectionClient::listAccepted($me); 200 → append each
    `user_id`; non-200 or GuzzleException → `$degraded[]='connections'` (own-posts-only). unique.
  STEP 2 posts+counts (HARD dep / spine): FeedClient::timeline($authorIds,$me,$limit); non-200 →
    `Json::raw($res, decode, status)` (fail the feed — feed-service is the only hard dependency).
  STEP 3 repost originals (NO N+1): collect `repost_of` ids (int>0, unique) from STEP 2 rows; if any →
    ONE FeedClient::getPosts($ids); 200 → map originals by id; non-200 → `$degraded[]='reposts'`.
  STEP 4 author enrichment (ONE batch): collect author_id of posts ∪ author_id of originals → ProfileClient::batch;
    on 200 build `id→allowlisted card` map; on non-200/GuzzleException → `$degraded[]='profiles'` (cards null).
  STEP 5 assemble (preserve feed-service newest-first order — map in place, NEVER re-sort by a map, Pitfall 6):
    each post gets `author` (allowlisted card|null); if repost_of>0 and original present, attach
    `original` (the original + its allowlisted `author`); else `original:null` (deleted original — Pitfall 5).
  Emit `Json::list($res,$out, $degraded? ['degraded'=>true,'parts'=>array_values(array_unique($degraded))] : [])`.

    output, so wrap them in ONE `GuzzleHttp\Promise\Utils::settle([...])->wait()` for the visible parallel
    fan-out slide (use FeedClient::getPostsAsync + ProfileClient::batchAsync). Resolve order is irrelevant.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → gateway (JWT) | JWT verified at gateway; user_id from the JWT-set attribute, never the body |
| gateway → feed-service | gateway maps JWT user_id → trusted X-User-Id; feed-service enforces owner-scope |
| upstream profile body → client | /users?ids= includes email; gateway MUST allowlist before emitting |
| feed-service → gateway (repost original) | a deleted/missing original must degrade to null, never crash the feed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Information disclosure (PII) | FeedController enrichment (feed/comment/single-post) | mitigate | `array_intersect_key($u, array_flip(['id','username','display_name','avatar_url']))` on every author card; verify asserts no `'email'` literal (Pitfall 2). |
| T-04-14 | Spoofing/Elevation (IDOR) | mutation passthrough | mitigate | caller from JWT `getAttribute('user_id')` via me(); FeedClient injects it as X-User-Id; body user_id never trusted. |
| T-04-15 | Denial (availability) | feed() degrade | mitigate | connection-service down → own-posts + `meta.degraded:['connections']`; profile/reposts failures degrade to null cards/original; feed-service is the only hard dep (Pitfall 7). |
| T-04-16 | Tampering/Integrity (N+1 + ordering) | feed() STEP 3/5 | mitigate | repost originals batch-resolved in ONE getPosts call (no N+1, Pitfall 5); order preserved by mapping in place (Pitfall 6). |
| T-04-17 | Info disclosure (deleted original) | feed()/showPost original resolution | mitigate | missing original id in the batch map → `original:null` (null-guard), never a 500 (Pitfall 5). |
| T-04-18 | Spoofing | /api/feed auth | mitigate | `/feed` + mutations require `$jwtMw` (no anonymous feed; my_reaction is viewer-relative); single-post uses `$optMw` (anon → my_reaction null). |
