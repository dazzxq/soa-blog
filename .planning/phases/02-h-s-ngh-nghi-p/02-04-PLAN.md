<!-- Existing frontend helpers (web/assets/app.js) — use directly. -->
- window.api.get/post/patch/delete(path[, body]) → returns {ok,status,data,meta}; auto-attaches Bearer if logged in.
- window.auth.token()/user()/isLoggedIn()/save()/logout()
- window.formatDate(iso) → vi-VN
- window.navbar() → Alpine data for the top bar (used by existing pages)
- Pages load app.js as a classic script BEFORE Alpine CDN, then declare <body>/<main x-data="...">.

Composition response shape (from Plan 03):
GET /api/profiles/{id}/full → { data: { id, username, display_name, avatar_url, cover_url, headline,
  location, about, created_at, experience:[{id,company,title,start_date,end_date,description}],
  education:[{id,school,degree,field,start_year,end_year}], skills:[{id,name}], connection_status },
  meta?: { degraded:true, parts:[...] } }

CRUD (JWT required, owner only):
  PATCH  /api/profiles/me                  {display_name?,avatar_url?,cover_url?,headline?,location?,about?}
  POST   /api/profiles/me/experience       {company,title,start_date,end_date?,description?}
  PATCH  /api/profiles/me/experience/{id}  DELETE same
  POST   /api/profiles/me/education         {school,degree?,field?,start_year?,end_year?}
  PATCH  /api/profiles/me/education/{id}    DELETE same
  POST   /api/profiles/me/skills            {name}    DELETE /api/profiles/me/skills/{id}
Resolve own id: api.get('/me') → data.id.
