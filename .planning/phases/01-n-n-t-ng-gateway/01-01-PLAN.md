Create `scripts/smoke-phase1.sh` as an executable bash script (`#!/usr/bin/env bash`, `set -euo pipefail`). Base URL constant `GW="${GW:-http://127.0.0.1:8000}"`. The script runs against an already-running stack (caller does `docker compose up -d --wait` first). Implement these checks, each printing a `[smoke] PASS/FAIL <name>` line and incrementing a failure counter; exit 1 at the end if any failed:

1. CONTAINER COUNT (PLAT-05): `count=$(docker compose ps --services | wc -l | tr -d ' ')`; assert `count -le 9`. Print the count.
2. PER-SERVICE HEALTH (PLAT-01/05): loop `for s in profile connection feed search notification; do docker compose exec -T gateway wget -qO- "http://$s-service/health"; done`; for each assert the JSON contains `"db":"ok"` (use `grep -q '"db":"ok"'`). Fail the named service if not.
3. GATEWAY FAN-OUT (PLAT-01/D-10): `body=$(curl -sf "$GW/api/health")`; assert `echo "$body" | grep -q '"status":"ok"'` AND that all 5 service keys appear: loop `for s in profile connection feed search notification; do echo "$body" | grep -q "\"$s\""; done`.
4. PROFILE ROUTE (PLAT-02): `curl -sf "$GW/api/profiles/2"` and assert output contains `duyet` (account id 2). `grep -q duyet`.
5. AUTH GUARD (PLAT-03): `code=$(curl -s -o /dev/null -w '%{http_code}' "$GW/api/me")`; assert `code` equals `401`.
6. REQUEST-ID DOWNSTREAM RECEIPT (PLAT-04 / D-12 — ISSUE-5, the concrete downstream proof, NOT just a response-header presence check):
   - Issue ONE request to the gateway capturing BOTH headers and body: `resp=$(curl -s -D /tmp/smoke-hdr.txt "$GW/api/health")` (the gateway generates an X-Request-Id, forwards it to each stub, and each stub echoes it back as `rid` in its /health body; the gateway surfaces those bodies under `services.<svc>`).
   - Extract the gateway response header value: `rid=$(grep -i '^x-request-id:' /tmp/smoke-hdr.txt | tr -d '\r' | awk '{print $2}')`. Assert `rid` is non-empty (gateway set the response header).
   - PROVE DOWNSTREAM RECEIPT: assert the SAME `rid` appears inside the gateway /api/health BODY (i.e. a downstream stub received the forwarded id and echoed it as `rid`): `echo "$resp" | grep -q "\"rid\":\"$rid\""`. If the body does not contain `"rid":"$rid"`, the gateway did NOT forward the id downstream — FAIL with a clear message `[smoke] FAIL request-id-downstream (gateway X-Request-Id=$rid not echoed by any service)`.
   - This single assertion replaces a bare header-presence check: it proves (a) the gateway sets X-Request-Id on its response AND (b) the same id reached a downstream service (D-12), aligning Plan 01's assertion with Plan 04's claim.
7. LOGIN ALL 5 ACCOUNTS (PROF-01/D-06): loop `for u in demo duyet long diep tai; do curl -sf -XPOST "$GW/api/auth/login" -H 'Content-Type: application/json' -d "{\"login\":\"$u\",\"password\":\"demo@123**\"}" | grep -q '"token"'; done`. Fail the named account if no token returned.

End with: if failures==0 print `[smoke] ALL PASS` and exit 0, else print `[smoke] <n> FAILURES` and exit 1. `chmod +x scripts/smoke-phase1.sh`.

Do NOT install any framework — bash + curl + docker compose only. Use the password literally `demo@123**` (matches db/99-seed.sql bcrypt hash).
