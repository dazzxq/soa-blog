Cách inject (no-build, no innerHTML cho dữ liệu người dùng): proNav trả về MỘT chuỗi HTML TĨNH (chỉ chứa markup Alpine `x-data/x-text/x-if`, KHÔNG nội suy dữ liệu user vào string), gắn x-data lên `<div id="pronav">`, RỒI khởi tạo subtree bằng `Alpine.initTree` để reactivity hoạt động bất kể proNav chạy trước hay sau Alpine init. Mẫu cụ thể:

```js
window.proNav = function (active) {
  // active: 'feed' | 'connections' | 'search' | 'profile' | '' (để tô đậm link hiện tại)
  var el = document.getElementById('pronav');
  if (!el) return;
  el.setAttribute('x-data', 'proNavData("' + (active || '') + '")');
  el.setAttribute('x-init', 'init()');
  el.innerHTML = PRONAV_HTML;   // markup TĨNH — mọi dữ liệu user render qua x-text bên trong, KHÔNG nội suy chuỗi
  // QUAN TRỌNG: Alpine 3 chỉ quét DOM MỘT LẦN tại init. x-data gắn SAU init sẽ KHÔNG được tự quét.
  //   - Nếu proNav chạy SAU khi Alpine init (Alpine đã sẵn): phải tự khởi tạo subtree này.
  //   - Nếu proNav chạy TRƯỚC khi Alpine init: walk init của Alpine sẽ tự nhặt x-data (initTree chưa có → bỏ qua an toàn).
  if (window.Alpine && typeof Alpine.initTree === 'function') {
    Alpine.initTree(el);
  }
};
```

KHÔNG dựa vào thứ tự gọi để có reactivity — `Alpine.initTree(el)` làm proNav an toàn cho CẢ hai trường hợp (gọi sớm pre-Alpine và gọi muộn post-Alpine). Sửa mọi comment cũ kiểu "gọi lại an toàn vì chỉ set attribute + innerHTML" — điều đó KHÔNG đủ để có reactivity; chính `initTree` mới đảm bảo.
